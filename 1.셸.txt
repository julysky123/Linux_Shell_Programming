셸
  키보드로부터 입력한 명령어를 운영체제에게 전달하여 그 명령어를 실행하게 하는 프로그램.
  사전적 의미로 껍데기를 의미하며 이는 사용자와 운영체제 내부 사이의 인터페이스를 감싸기 때문.

셸은 크게 2가지로 나뉜다.
	1. CLI (Command Line Interface): 일반적인 셸을 의미.
 	2. GUI (Graphic User Interface): 그래픽을 사용한 인터페이스

셸의 종류
	1. bourne shell (unix에서 사용하던 대표적인 프로그램 -> bourne-again shell(bash)
	2. tcshell
	3. zshell ...

<shell prompt>
  -> 사용자로부터 입력을 대기하고 있다는 표시
  -> 운영체제마다 다소 다르지만 대부분 대동소이.

linux@ubuntu:~$
  -> 사용자이름@호스트이름 : 작업중인디렉토리전체경로명$

<명령어를 실행하는 방법>
  1. 하나의 명령어를 실행하는 방법.
    linux@ubuntu:~$ date
    linux@ubuntu:~$ cal
  2. 다수의 명령어를 실행하는 방법. (세미콜론 사용)
    linux@ubuntu:~$ pwd; date; cal
  3. 화면 지우기. clean, ctrl+l 

<과거에 입력한 명령어를 찾는 방법(히스토리)>
  위, 아래 방향키.

<자동 완성 기능>
  프롬프트 상에서 명령어나 파일 이름의 일부분을 입력한 다음 탭 키를 누르면 나머지 부분이 완성된다.
  
<종료>
  exit
  
<리눅스 파일 시스템 구조>
  ->리눅스 : 단일 파일 시스템을 사용.
  ->윈도우즈 : 저장 장치마다 개별 파일 시스템.
  
<디렉토리>
  ->윈도우즈의 폴더 개념
  ->파일과 디렉토리를 포함하고 있는 개념

<작업 디렉토리>
  ->사용자가 현재 작업하고 있는 디렉토리
  ->ls로 디렉토리 내 파일 확인 가능.
  
<작업 디렉토리를 확인하는 방법>
  -> pwd (print working directory)
  -> / : root directory.
-> 사용자가 리눅스에 로그인을 하면 사용자의 홈 디렉토리가 작업 디렉토리가 된다.
-> 홈 디렉토리는 사용자에게 파일 쓰기 권한이 부여된 유일한 공간이다.


<ls>
-> 디렉토리의 내용을 출력하는 명령어
-> -l: 현재 디렉터토리의 내용을 자세하게 출력. 
-> -a: 숨김파일도 출력. (파일이름 앞에 .)
-> 옵션은 두개이상 동시에 사용해도 되고 순서를 바꿔도 된다. (ex: -la)
-> ls 디렉토리명 : 틀정 티렉토리의 내용 확인.
-> ls -l 파일명 : 해당 파일의 정보가 출력.
-> ls -l 하위디렉토리 : 하위디렉토리 아래의 파일들을 자세히 출력.
-> ls -ld 하위디렉토리 : 하위디렉토리의 정보가 자세히 출력.
-> ls -F 디렉토리인 경우 이름의 끝에 /를 추가.
-> ls -r 출력 결과를 역순으로 표시.
-> ls -R 하위디렉토리를 순회하면서 그 안의 내용을 출력.
-> ls -t 파일의 수정 시간을 오름차순으로 정렬하여 출력.

<cd>
 -> . 현재 경로
 -> .. 상위 경로
 -> cd 만 적으면 최상위 경로로
 -> cd~ 최상위 경로
 -> ./는 생략이 가능.
 -> cd - 이전작업 디렉토리로 이동.

-참고-
ls 명령어도 절대 경로와 상대 경로 모두 사용할 수 있다.
